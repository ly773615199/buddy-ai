Buddy AI 产品需求方案

视觉形象系统设计


一、产品概述

产品名称： Buddy


产品定位： 具备陪伴属性、有成长潜力的AI助手


核心理念： Buddy不是用户设计的工具，而是因用户而成长的伙伴。每个Buddy都是独一无二的，它的形象是用户习惯与性格的"外化映射"。



二、视觉形象系统

2.1 设计原则

原则	说明
千人千面	每个用户的Buddy形象都独一无二
渐进成长	形象随使用逐步清晰，不是一步到位
无感塑造	用户不需要主动"设计"，Buddy在使用中自然成形
可控边界	用户有参与感，但不承担设计负担

2.2 成长阶段体系

Buddy的成长分为四个阶段，核心逻辑为"能量凝聚"——从混沌光团逐渐凝聚为清晰形象。


阶段	形态	视觉表现	用户感知
蛋	能量混沌态	缓慢呼吸的光团，看不出具体形态	潜力、未知、期待
孵化	轮廓初现	光团中有模糊形状隐约浮现	开始回应、有了存在感
成长	细节渐显	轮廓清晰，细节逐步填充	逐渐了解、建立连接
成形	完全体	用户Buddy的完整形象呈现	深度陪伴、独一无二

2.3 蛋阶段的用户参与

蛋阶段不完全是空白，保留用户选择的种子痕迹：


维度	选项	对蛋的影响
主色调	用户选1-2个色	蛋的光色
质感倾向	柔软 / 通透 / 锋利 / 温润	蛋的形状和边缘
气质方向	温暖 / 冷静 / 活泼 / 神秘	光的节奏和明暗

2.4.2 Prompt各组成部分来源

组成部分	数据来源	收集方式
产品风格框架	产品预设	固定不变，保证风格统一
用户选择	注册时的关键词选择	选择题，1-2个词
用户行为数据	使用习惯分析	后台自动采集，无需用户感知
随机种子	注册时间+生日+随机数	自动生成，保证唯一性且可复现
阶段约束	当前成长阶段	系统自动附加
质量词	产品预设	固定不变

2.4.3 用户行为数据对Prompt的影响

用户行为特征	对形象的影响方向
经常深夜使用	偏暗色调、安静气质
喜欢高效对话	形象利落、简洁线条
喜欢闲聊	形象圆润、柔和质感
高频使用	细节更丰富、更精致
偏好理性话题	几何感更强
偏好感性话题	有机形态更多


三、形象生成技术方案

3.1 两阶段策略

第一阶段：MVP — 预设组件库方案

text
text
形象 = 头部组件 + 身体组件 + 特征组件 + 颜色方案

提供数十个部件，排列组合
质量可控，风格统一
生成速度快，算力成本低
适合验证产品逻辑和用户接受度

第二阶段：V2 — AI实时生成

接入图像生成模型（如Stable Diffusion）
基于prompt实时生成
质量更高，形象更丰富
需要增加筛选和审核机制

备选：混合方案

预设骨架保证结构和动效适配
AI填充纹理、色彩和细节

3.2 动效适配方案

不管生成什么形象，都需要能动起来：


方案	说明	适用阶段
统一骨骼绑定	所有形象共用一套骨骼系统	预设组件方案
2D立绘 + 通用动效	生成静态立绘，动效用呼吸/浮动/光效	AI生成方案
Live2D	2D图像+骨骼变形，成本适中	中期过渡

text

text

  ↓  使用积累

4.3 成长中的用户互动

时机	交互方式	目的
成长阶段	偶尔询问"你喜欢我这样吗？"	给用户参与感
成形阶段	展示形象 + 询问满意度	收集反馈
成形后	提供微调选项	允许个性化修正


五、边界控制与安全

5.1 形象生成约束

约束项	说明
风格边界	只允许非写实风格，避免恐怖谷效应
安全过滤	自动过滤色情、暴力、政治敏感内容
质量筛选	生成后自动评分，低于阈值重新生成
兜底方案	提供"重置为默认形象"选项

5.2 用户控制权

操作	阶段	限制
重新随机	蛋阶段	最多1-3次
微调	成形后	仅限颜色、大小、细节
重置	任意阶段	可回到默认形象


六、产品路线图

Phase 1：MVP（验证期）

 3-5个预设组件库，排列组合生成形象
 四个成长阶段的通用动效（光团呼吸→凝聚→清晰→成形）
 注册时的选择题交互流程
 基础的阶段成长逻辑

Phase 2：V2（体验升级）

 接入AI图像生成模型
 用户行为数据影响形象生成
 扩展组件库或开放AI生成
 成形后的微调功能

Phase 3：V3（生态扩展）

 十二生肖等文化主题扩展包
 Buddy形象社交展示（用户之间可见）
 Buddy形象商城/特殊活动形象
 3D形象和AR展示


七、关键设计决策总结

决策点	结论	理由
形象由谁决定	Buddy因用户而成长	陪伴者定位，不是工具
用户参与方式	选种子，不设计	降低负担，保留参与感
成长逻辑	能量凝聚	适配所有形象，叙事统一
生成方案	MVP预设组件，V2接入AI	先验证逻辑，再提升质量
形象来源	用户行为+随机种子+用户选择	独一无二且可复现
