## 一、Buddy 的终态愿景

**Buddy 不需要基座大模型，只需要一个个独立运行的领域专家微模型。**

```
传统架构:                            Buddy 终态:
┌──────────────┐                    ┌─────────────────────────────┐
│ 基座大模型    │ ← 4GB+ 固定       │ 没有基座                     │
│ (不可变)      │                   │                             │
├──────────────┤                   │  ┌─────┐ ┌─────┐ ┌─────┐   │
│ LoRA 适配器   │ ← 叠加插件        │  │Go专家│ │法务  │ │财务  │   │
│ × N 个领域    │                   │  │40MB │ │100MB│ │20MB │   │
└──────────────┘                   │  └─────┘ └─────┘ └─────┘   │
                                   │                             │
用户买基座，选装插件                  │  每个是独立自包含小模型        │
                                   │  持续成长，按需安装           │
                                   │  全部三进制，纯 CPU 推理      │
                                   └─────────────────────────────┘
```

## 二、全链路管线

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 知识积累  │    │ 训练触发  │    │ 微调训练  │    │ 蒸馏     │    │ 安装     │    │ 推理     │
│          │───▶│          │───▶│          │───▶│          │───▶│          │───▶│          │
│ 对话→STMP │    │ 100条知识 │    │ LoTA-QAF │    │ 大→小    │    │ 商城下载  │    │ 本地运行  │
│ 判断力样本 │    │ trainable │    │ 三进制LoRA│    │ 三进制   │    │ 秒装     │    │ 纯CPU    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     ↑                                                                           │
     └───────────────────── 增量反馈闭环 ←─────────────────────────────────────────┘
                              模型在成长
```

### 各环节详解

**① 知识积累（对话 → STMP）**

- 用户日常对话中提取专业知识、判断力模式、纠正样本
- 存入 STMP 知识图谱，按领域分类
- 领域画像跟踪成长阶段：seed → sprout → growing → trainable → mature
- 三进制在此环节：不适用（数据是文本）

**② 训练触发（trainable 阶段）**

- 领域知识达到 100 条 + depthScore > 0.5 → trainable
- 不再等 500 条 mature 门槛（改进前需 100+ 天，改进后 10-20 天）
- 预检机制过滤无效对话（避免 75% 空提取浪费）

**③ 微调训练（LoTA-QAF 三进制 LoRA）**

- 云端 GPU 执行，基于 GPTQ 量化基座模型
- 三进制适配器 A∈{-1,0,1}，B 通过 t-SignSGD 优化
- 无损合并到量化权重，训练后适配器仅 128 KB（4096×4096, rank=64）
- 三进制在此环节：**训练时就引入三进制约束，不是训练后再量化**

**④ 蒸馏（大模型 → 三进制小模型）**

- 将大模型+LoRA 的能力蒸馏到独立小模型（100M - 1B 参数）
- BitNet 风格三进制训练，模型权重全部 {-1, 0, 1}
- 产物大小：100M 参数 ≈ 20 MB，1B 参数 ≈ 190 MB
- 三进制在此环节：**核心战场，决定最终体积和推理速度**

**⑤ 安装（商城分发）**

- 三进制格式极小，190 MB 的 1B 模型 vs 500 MB 的 4-bit 模型
- 商城秒装，弱网可用，离线传输可行
- 三进制在此环节：**体积缩小 2.6x，分发体验质变**

**⑥ 推理（本地运行）**

- 权重 {-1, 0, 1}，乘法变加法，纯整数 ALU
- 不需要 GPU / FPU / TPU，任何 CPU 都能跑
- 手机推理 10-20 tok/s，树莓派可用
- 三进制在此环节：**推理速度提升 16x（vs FP16），边缘质变**

## 三、增量成长机制

微模型不是训练一次就定型的，它跟着用户一起长大。

### 成长时间线

```
第 1 月 ──→ 第 3 月 ──→ 第 6 月 ──→ 第 12 月
 50条        300条        800条       2000+条
 100M参数     200M参数     500M参数     1B参数
 20 MB        40 MB        100 MB      190 MB
 ★★☆☆☆      ★★★☆☆      ★★★★☆     ★★★★★
 萌芽         成长中       成熟        专家
```

### 增量模式

**模式 A：模型变大（参数量增长）**

随知识量增加，扩大模型容量。适合知识面广的领域。

```
100M → 200M → 500M → 1B
20MB → 40MB → 100MB → 190MB
```

**模式 B：模型变深（知识密度增长，参数量不变）**

模型大小不变，但三进制权重中 0 的比例降低——"空位"被知识填满。适合知识深度增长的领域。

```
初始: {-1, 0, 1} 中 70% 是 0 → 浅层知识
3月后: 50% 是 0 → 有经验模式
6月后: 30% 是 0 → 深度判断力

体积不变 (同 200M 参数)，但有效参数越来越多
```

**模式 B 更优雅**——三进制天然支持：{-1, 0, 1} 中 0 的比例就是"未使用容量"。

### 夜间自动增量

```
用户睡觉时自动执行:

  今日对话 → STMP 新增 5 条知识
    ↓
  预检: 是否有专业知识信号? (containsExpertKnowledge)
    ↓
  有价值 → 追加训练 10 步 t-SignSGD
    ↓
  少量权重 0 → +1 或 -1 (模型在"长肉")
    ↓
  合并到本地模型 → 明天用户对话时已更新
    ↓
  耗时: 几分钟 (纯 CPU, 三进制整数运算)
  体积: 不变 (模式 B) 或微增 (模式 A)
```

## 四、三进制量化技术选型

### 4.1 训练端：LoTA-QAF

**Paper**: [arXiv:2505.18724](https://arxiv.org/abs/2505.18724) (NeurIPS'25 Poster)

- 三进制适配器对齐量化网格，直接调整量化权重
- t-SignSGD 优化器：动态百分位阈值 + 符号梯度下降
- 无损合并：合并后回到纯 4-bit，零额外开销
- 实验：Qwen-2.5 14B 2-bit 上比 16-bit LoRA 高 5.14%
- 推理速度：合并后比 LoRA 快 1.7x-2.0x
- 代码：MIT License，github.com/KingdalfGoodman/LoTA-QAF

### 4.2 模型端：BitNet 1.58-bit

**Paper**: [arXiv:2411.05882](https://arxiv.org/abs/2411.05882)

- 基座模型权重全部三进制 {-1, 0, 1}（1.58 bit/weight）
- 在 MLP、GNN、Transformer 上与 FP32/FP16 持平或更好
- 推理只需整数加法，无需浮点单元
- 为三进制独立小模型提供理论基础

### 4.3 其他参考

| 方法 | 核心贡献 | 与 Buddy 的关系 |
|------|---------|----------------|
| IntLoRA (ICML'25) | 整数型 LoRA，训练中保持量化 | 训练端参考 |
| LoRAQuant | 混合精度超低比特量化 LoRA | 多适配器管理参考 |
| QA-LoRA (ICLR'24) | 无损合并适配器到量化权重 | 合并机制参考 |

## 五、PoC 验证结果

已实现纯 NumPy PoC（`LoTA-QAF/poc_ternary_lora.py`），验证核心机制：

```
配置: 4096×4096, 4-bit 基座, rank=64 三进制适配器

✅ 三进制适配器: A∈{-1,0,1} 正常生成
✅ 阈值化: 9.78% 权重被调整，90.22% 稀疏
✅ 无损合并: 合并后 W_int 仍在 [0,15]，纯整数
✅ 偏移因子: zero_plus 计算正确
✅ 体积压缩: 128 KB vs 1 MB (8x 倍)
```

推理误差比偏高（22.83x）是因为 B 矩阵随机初始化、未经过实际训练，属预期结果。机制验证完全通过。

## 六、三进制在各环节的价值排序

```
优先级   环节           价值点                        收益
────────────────────────────────────────────────────────────
P0      推理速度        乘法→加法，纯整数 ALU          16x 提速
        (终极价值)      手机 10-20 tok/s               边缘质变

P0      模型蒸馏        大模型→三进制小模型            190MB vs 500MB
        (核心战场)      决定最终体积和推理方式          一切的前提

P1      存储/分发       三进制紧凑格式                  商城秒装
                       190MB 模型                     CDN 成本降 2.6x

P1      增量训练        t-SignSGD 低成本               每晚 CPU 几分钟
                       0→±1 的翻转                    模型持续成长

P2      训练微调        LoTA-QAF 三进制 LoRA           训练时引入约束
        (必要前提)      需要 GPU + Triton              为蒸馏铺路

—       知识积累        不适用（数据是文本）            —
        (STMP 对话)
```

## 七、集成路径

### Phase A：定义格式 + 存储层（1 周）

- 定义 `.ta` 三进制模型格式规范
- 实现三进制权重打包/解包（2-bit packing）
- 实现权重体积计算和元数据管理
- 改造 `src/lora/service.ts` 支持 `.ta` 格式
- **交付物**: 可存储、可列出、可删除三进制模型

### Phase B：推理合并 + 本地运行（2 周）

- 实现本地 `merge_markers_to_weights`（纯 TS/JS）
- 实现 `zero_plus` 偏移合并
- 推理引擎：三进制权重 → 整数加法推理
- 性能基准：CPU 推理速度 vs FP16 / INT4
- **交付物**: 三进制模型可本地推理，纯 CPU

### Phase C：增量训练 + 自动成长（2 周）

- t-SignSGD 优化器实现（纯 TS 或调用 Python）
- 夜间自动增量训练流程
- 领域成长阶段联动（trainable → 自动触发训练）
- 模式 A/B 切换策略
- **交付物**: 模型夜间自动成长，用户无感

### Phase D：蒸馏 + 独立模型（3 周）

- 知识蒸馏管线：大模型 LoRA → 三进制小模型
- BitNet 风格三进制训练
- 模型架构选择（轻量 Transformer）
- 评估体系：领域准确率 vs 模型大小
- **交付物**: 独立三进制专家模型，不依赖基座

### Phase E：商城 + 生态（2 周）

- 商城三进制模型上架/分发
- 体积对比展示（vs 标准量化模型）
- 用户体验：一键安装、成长进度展示
- 模型分享/交易机制
- **交付物**: 完整商城生态

## 九、技术风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 三进制推理精度不够 | 专家模型质量差 | 模式 B 渐进验证，初期保留 API 兜底 |
| 蒸馏效果不达预期 | 小模型能力弱 | 多架构对比（1B/3B/不同层数） |
| 增量训练灾难性遗忘 | 学了新的忘了旧的 | 经验回放 + 定期全量微调 |
| t-SignSGD 收敛慢 | 增量训练耗时长 | 调整阈值调度策略 |
| 三进制硬件兼容性 | 部分 CPU 架构不优 | 提供 INT8 回退路径 |
