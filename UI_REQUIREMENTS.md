# Buddy UI 改造方案 v2

**日期**: 2026-04-23
**来源**: 代码审计 + 行业调研 + 深度讨论
**核心理念**: 光灵是独立生命体，UI 是中性工具面板

---

## 一、设计哲学

### 光灵 ≠ 装饰，光灵 = 独立存在的生命体

```
错误理解：光灵是 UI 的一部分，情绪影响整个界面
正确理解：光灵是独立实体，UI 是它住的房间

  光灵（伙伴）：有情绪、有表情、有声音、有成长
  UI（工具）：对话、工具、记忆、设置 — 功能性的，中性的

  类比：
    光灵 = 你桌上的猫
    UI   = 你的电脑屏幕
    猫开心不会让你的屏幕变粉色
    但猫跳到键盘上你会注意到它
```

### 情绪表达的边界

```
情绪只通过光灵自身表达：
  ✅ 粒子形态变化（收缩/扩散/旋转/跳跃）
  ✅ 颜色/亮度变化
  ✅ 运动节奏（快/慢/颤抖/脉冲）
  ✅ 眼睛状态（睁大/眯起/闭合）
  ✅ 声音（它自己的声音）

情绪不侵入 UI：
  ❌ 气泡颜色不随情绪变
  ❌ 背景色不随情绪变
  ❌ 通知语气不随情绪变
  ❌ 数据措辞不随情绪变
```

### 光灵的三个存在层次

```
Layer 1: 界面内（现在）
  光灵在网页的 280px 区域里
  用户打开页面才能看到

Layer 2: 桌面独立体（近期目标）
  Electron/Tauri 浮窗，透明背景
  光灵在屏幕上自由移动
  感知窗口/剪贴板/文件/时间/网络
  可以主动通知

Layer 3: 环境智能体（远期目标）
  理解屏幕内容
  语音自然对话
  多设备同步
  多光灵社交
```

## 二、必要面板（UI = 中性工具）

### 1. 💬 对话面板（保留，核心）
- 消息列表 + 输入栏
- Agent 执行轨迹可视化（思考→工具→观察→回答）
- 流式输出 + Markdown 渲染

### 2. 🔧 工具面板（新增）
- 工具列表（名称 + 描述 + 来源：内置/MCP/Skill）
- 每个工具的调用次数 + 成功率
- 最近 10 次执行记录（时间 + 参数 + 结果摘要 + 耗时）
- MCP Server 连接状态

### 3. 🧠 记忆面板（新增）
- 领域知识树（domain → 节点列表）
- 每个节点：内容 + 类型 + 置信度 + 访问次数
- 手动搜索 + 编辑 + 删除
- 记忆容量统计
- STMP 检索浏览

### 4. 📊 活动面板（新增）
- 操作时间线（消息/工具调用/状态变更）
- Token 用量趋势（按天）
- 预估成本
- 环境信息（位置/网络/设备）
- 梦境日志
- 传感器数据（从"传感"tab 合并）

### 5. ⚙️ 设置面板（新增，最高优先级）
- LLM 配置（Provider/Model/API Key/BaseURL + 测试按钮）
- 行为偏好（回复风格/语言/工具确认策略）
- 外观（主题色/暗亮切换/字体大小）
- 平台管理（Telegram/Discord 连接状态）
- 数据管理（导出/清除/重置）
- 光灵设置（声音开关/通知频率/桌面模式开关）

### 6. 🗺️ 探索面板（保留）
- 养成展示（5 维属性/进化阶段/稀有度）
- 功能探索图谱
- 活动热力图（需改为真实数据）

### 7. 👁️ 视觉面板（保留，补配置入口）
- 摄像头/OCR/场景分析
- 需要增加 API Key 配置入口

### 8. 🎓 专家面板（保留）
- 三进制专家商城
- 安装/卸载/训练进度

### Tab 重组

```
当前 6 tab → 目标 8 tab:

  💬 对话      → 保留
  🔧 工具      → 新增
  🧠 记忆      → 新增
  📊 活动      → 新增
  ⚙️ 设置      → 新增
  ──────────
  🗺️ 探索      → 保留
  👁️ 视觉      → 保留
  🎓 专家      → 保留

  移除/合并:
  📡 传感      → 合并到活动面板
  🧠 认知      → 拆分到记忆 + 工具 + 活动
```

## 三、光灵独立化（核心差异化）

### 3.1 桌面浮窗模式

```
技术路径：Electron 透明窗口 + Pixi.js 渲染

实现：
  - 透明无边框 BrowserWindow
  - 光灵在屏幕任意位置渲染
  - 鼠标穿透（点击光灵才响应，其他区域穿透到下层窗口）
  - 窗口层级：alwaysOnTop（可配置）
  - 多显示器支持

用户操作：
  - 双击光灵 → 打开对话面板
  - 右键光灵 → 快捷菜单（设置/隐藏/对话/工具）
  - 拖拽光灵 → 移动位置
  - 点击光灵 → 触发反应
```

### 3.2 自主行为系统

```
光灵不依赖用户操作，有自己的"生活"：

 idle 行为（已有，需扩展）：
  - 在屏幕上缓慢飘移
  - 偶尔做出随机动作（闪烁/旋转/变色）
  - 靠近窗口边缘时"折返"
  - 长时间无操作 → 变小、变暗、进入"睡眠"

 环境感知行为：
  - 靠近代码编辑器 → 粒子变"代码绿"
  - 靠近浏览器 → 粒子变"搜索蓝"
  - 窗口最小化 → 光灵"失去依靠"，飘到屏幕中央
  - 窗口最大化 → 光灵"被挤到边上"
```

### 3.3 感知能力

```
所有感知能力基建已有或 API 原生支持：

 剪贴板感知（navigator.clipboard + 事件）：
  - 复制代码 → 光灵"分析"姿态
  - 复制 URL → 光灵"想去看看"姿态
  - 复制错误信息 → 光灵"我来修"姿态
  - 长时间不复制 → 光灵无聊飘动

 文件系统感知（已有 fs-watcher.ts）：
  - 新文件下载 → 光灵"发现新东西"
  - 文件修改 → 光灵"注意到变化"
  - 项目目录变化 → 光灵"整理"姿态

 时间感知（Date + 定时器）：
  - 早上 → 光灵阳光色、活跃
  - 深夜 → 光灵变暗、安静、打瞌睡
  - 午饭时间 → 光灵"饿了"姿态
  - 下班时间 → 光灵"想走"姿态

 网络感知（navigator.onLine + Network API）：
  - 网断了 → 光灵"被困住了"，粒子被框住
  - 网速慢 → 光灵移动变慢
  - 连上 VPN → 光灵"换了个地方"

 屏幕内容理解（截图 + 多模态 LLM，已有 omni.ts）：
  - 用户在看代码报错 → 光灵"着急"
  - 用户在看有趣内容 → 光灵"好奇"
  - 用户长时间不动 → 光灵"担心"
```

### 3.4 主动交互

```
光灵不是被动等用户来聊：

 系统通知：
  - 新邮件/消息 → 光灵闪烁、飘到屏幕角落
  - 日历事件快到 → 光灵做出"提醒"姿态
  - 工具执行完成 → 光灵做出"搞定了"姿态
  - 系统异常 → 光灵做出"警报"姿态

 主动对话：
  - 发现用户遇到困难 → 光灵主动弹出建议
  - 检测到重复操作 → 光灵"我可以帮你"
  - 长时间工作 → 光灵"该休息了"

 声音交互（已有 stts.ts + tts.ts）：
  - 光灵可以直接说话（不通过文字）
  - 语音唤醒 → 光灵"被叫到了"
  - 说话时粒子随声波振动
```

### 3.5 光灵社交

```
已有 FriendSystem + BuddyInteraction：

 好友光灵来访：
  - 好友的光灵来"串门"（粒子融合效果）
  - 光灵之间"对话"（气泡互发）
  - 一起完成任务 → 粒子协作效果

 排行榜：
  - 稀有度/进化阶段/使用天数
  - 光灵们"聚会"展示
```

## 四、形象 ↔ 能力联动

```
光灵的视觉反映它的真实能力：

 学会新 Skill → 粒子颜色变化/新增光效
 领域知识增长 → 对应颜色的粒子变多
 工具调用成功 → 粒子闪烁/光环扩散
 连续使用 → 粒子密度增加
 稀有度提升 → 光晕范围扩大
 进化阶段变化 → 粒子形态根本性改变
```

## 九、关键洞察

**光灵的价值不在于它"能做什么"，而在于它"在不在"。**

一个一直在那里的、有生命的、会反应的存在，
比一个功能强大的工具更有粘性。

行业在卷"AI 能做什么"，但没人卷"AI 长什么样"。
Buddy 的粒子系统是唯一一个实时渲染、状态驱动、可进化的 AI 形象 — 这是护城河。
