# 第三方处理者清单

> 最后更新：2026-05-10

## 一、概述

Buddy 采用**本地优先**架构，大部分数据处理在用户设备上完成。以下列出可能涉及数据传输的第三方服务。

所有第三方服务均为**用户可选**，可在设置中随时启用/禁用。

## 二、第三方服务清单

### 2.1 LLM API 服务

| 项目 | 内容 |
|------|------|
| 服务类型 | 大语言模型推理 |
| 可能的提供商 | OpenAI、DeepSeek、Anthropic、Google、Ollama（本地）、SiliconFlow、MiMo 等 |
| 传输数据 | 对话文本（不含图像帧） |
| 数据用途 | AI 推理和响应生成 |
| 用户控制 | 可选择本地模型（Ollama）完全避免云端传输 |
| 启用方式 | 用户在设置中配置 API Key 和端点 |

**数据不包含**：摄像头画面、麦克风音频、人脸信息、位置信息

### 2.2 语音识别服务

| 项目 | 内容 |
|------|------|
| 服务类型 | 语音转文字（STT） |
| 可能的提供商 | Edge TTS、OpenAI Whisper 等 |
| 传输数据 | 音频片段 |
| 数据用途 | 语音识别 |
| 用户控制 | 需用户主动开启麦克风 |
| 启用方式 | 用户在设置中启用语音功能 |

### 2.3 商城服务

| 项目 | 内容 |
|------|------|
| 服务类型 | 技能/能力包分享 |
| 传输数据 | 脱敏后的模型数据、技能代码、经验数据 |
| 数据用途 | 技能分享和下载 |
| 脱敏处理 | PII 扫描 + 人脸帧丢弃 + 用户名匿名化 |
| 用户控制 | 需用户主动发布 |

**脱敏规则**：
- 文本中的路径/IP/邮箱/token → 自动替换
- 含人脸的图像帧 → 发布时丢弃
- 用户名/ID → 替换为匿名 ID
- 对话记录中的个人信息 → 脱敏处理

## 三、数据传输原则

1. **用户同意**：所有第三方传输需用户明确启用
2. **最小化**：只传输必要的最少数据
3. **可撤回**：用户可随时关闭任何第三方服务
4. **透明**：隐私面板显示当前启用的第三方服务
5. **脱敏**：商城发布前强制脱敏处理

## 四、本地替代方案

| 功能 | 云端方案 | 本地替代 |
|------|---------|---------|
| AI 推理 | OpenAI/DeepSeek API | Ollama 本地模型 |
| 语音识别 | 云端 STT | 本地语音识别 |
| 技能分享 | 商城服务 | 本地导出/导入 |

用户可以选择完全本地运行，不与任何第三方服务交互。
